# 1교시 — 데이터 분석 개론 (이론 강화판) · 목차안

> **문제의식**: 기존 83장은 서사·후킹 위주 → 실제 강의 시 ~20분, 학생이 가져갈 '이론'이 부족.
> **해결**: 통계·데이터 보는 법·전체/나눠보기·기업 활용·AI를 본격 투입.
> **핵심 설계**: 낚시 3종을 각 이론 블록의 '도입 훅'으로 연결 → 재미(후킹)와 알맹이(이론)를 동시에.
>
> 분량 추정: 약 90~95장 / **약 65~75분** (현재 50분 초과 → 운영 방식은 아래 ‘페이스’ 참고)

---

## 오프닝 (약 5분)
- 표지: "1교시 — 데이터 분석, 왜 그리고 어디로"
- 강의 전체 체제(5교시, 이론 중심 + 실습 셀프스터디)
- 오늘의 여정(8부 지도)
- 훅: "오늘 여러분을 네 번 속입니다"

## Part 1. 왜 데이터 분석인가 (약 6분)
- 우리는 매일 '결정'한다 (개인·기업)
- 결정의 두 방식: 감 vs 데이터
- 감(직관)은 이렇게 틀린다 (편향·기억·규모)
- 감만 믿다 무너진 거인들 (코닥·블록버스터)
- 감 → 데이터 기반 의사결정으로
- **데이터 분석의 정의**: 데이터에서 패턴을 찾아 '결정에 쓸 답'을 만드는 일

## Part 2. 데이터란 무엇인가 — 보는 눈의 기초 (약 8분) 〔이론 보강〕
- 데이터 = 표: **행 = 관측치, 열 = 변수** (실제 표 예시)
- 정형 데이터 vs 비정형 데이터
- **변수의 종류**: 범주형(명목·순서) vs 수치형(이산·연속) ← *신규 이론*
  - 예: 성별/등급(범주) vs 키/매출(수치) — 변수 종류가 분석·차트를 결정
- 좋은 데이터 / 지저분한 데이터 맛보기 (결측·이상치 — 3교시 예고)
- "데이터를 본다"는 것의 의미: 숫자 더미 → 의미

## Part 3. 함정① 평균의 함정 → **기본 통계** (약 13분) 〔이론 핵심〕
- 낚시: "평균 연봉 5천만원" (제시→질문→예상→반전→해부)
- 그래서, 데이터를 '요약'하는 통계:
  - **중심값 3형제**: 평균 / 중앙값 / 최빈값 — 언제 무엇을 쓰나
  - **퍼짐(산포)**: 범위 / 분산 / 표준편차 — 직관적으로(평균만으론 부족)
  - **분포와 치우침**: 히스토그램으로 모양 보기, 한쪽으로 쏠림(skew)
  - 정리: "평균 하나가 아니라 중심 + 퍼짐 + 분포를 함께"
- 메시지: 숫자 하나(평균)에 속지 마라

## Part 4. 함정② 생존자 편향 → **데이터를 제대로 보는 법** (약 9분) 〔이론 보강〕
- 낚시: 2차대전 전투기 (제시→질문→반전→해부, 실화)
- 그래서, 데이터를 의심하는 법:
  - **표본과 편향**: 이게 '누구의' 데이터인가 (대표성)
  - **안 보이는 데이터 / 결측**: 없는 데이터가 더 많이 말한다
  - **상관 ≠ 인과**: 같이 움직인다고 원인은 아니다 ← *신규 이론(중요)*
    - 예: 아이스크림 판매↑ & 익사↑ → 진짜 원인은 '여름'
  - 데이터를 볼 때 던질 질문 체크리스트
- 일상 확장: 성공 수기·합격 후기·투자 후기의 함정
- 메시지: 보이는 것만 믿지 말고, 안 보이는 것을 생각하라

## Part 5. 함정③ 심슨의 역설 → **전체 보기에서 나눠 보기로** (약 13분) 〔이론 핵심〕
- 낚시: 1973 UC버클리 (제시→질문→반전→해부, 실화)
- 그래서, 데이터를 보는 2단계:
  - **1단계 전체 보기**: 요약통계로 큰 그림 잡기 (describe의 개념)
  - **2단계 나눠 보기**: 그룹별로 쪼개 비교(세분화 = groupby의 개념)
  - **시각화로 보기**: 막대·히스토그램·산점도·박스 — 언제 무엇을 (개념만)
  - 핵심 흐름: "전체 → 나눠 보기"가 인사이트를 만든다 (3교시 실습 직결)
- **낚시 3종 종합**: 평균에 속고 · 안 보여서 속고 · 안 나눠서 속는다 → 그래서 분석을 배운다

## Part 6. 기업은 데이터를 어떻게 쓰나 — **수집 → 처리 → 활용** (약 10분) 〔활용 보강〕
- **데이터 파이프라인** 도식: 수집 → 처리 → 분석 → 활용
- 〔데이터 엔지니어링 심화 — 대학생 대상, 깊게〕
  - 데이터 종류 ① 구조: 정형/반정형/비정형 (+저장: RDB·NoSQL·객체스토리지)
  - 데이터 종류 ② 속도: 배치/실시간(스트리밍)/이벤트드리븐 (지연시간)
  - 저장: OLTP(거래·행지향) vs DW/OLAP(분석·열지향)
  - 이동·변환: ETL/ELT (Transform = 3교시 정제와 동일)
  - 대용량: 병렬·분산 처리(맵리듀스·하둡·스파크)
  - 현대 아키텍처: 웨어하우스 vs 레이크 vs 레이크하우스
- 분야별 실제 활용: 마케팅 / 금융 / 의료 / 공공 / 스포츠 (각 구체 사례)
- 강사 현장(fintech): 비트코인·금값·부동산 — 차트 + 질문 (일화는 발표자 노트)
- 정리: 분야는 달라도 '질문 → 데이터 → 해석'은 같다

## Part 7. AI 시대의 데이터 분석 (약 9분) 〔현장·웹서치〕
- 도구의 진화 4단계 (압축): 엑셀 → 통계·BI → ML → LLM·에이전트
- **AI가 파이프라인을 어떻게 바꾸나**: 에이전트가 수집~리포트를 자동 시도
- **AI 활용 방식**: 자연어로 분석 시키기 / 자동 리포트 / 이상탐지 (인식 수준)
- **입문 분석가의 역할 이동**: 추출·실행 → 검증·해석·판단 (출처: 2026 현장)
- **AI는 자신 있게 틀린다**: ChatGPT "3배" vs 실제 "2.6배" → 검증의 중요성 (낚시와 호응)

## Part 8. 닫기 (약 4분)
- 속지 않는 사람의 두 가지 힘: 좋은 질문 + 검증·해석
- 핵심 문장: "도구는 변해도, 좋은 질문을 던지고 결과를 해석하는 힘은 사람의 몫"
- "오늘 네 번 속았다" 콜백 (평균·생존자·심슨·AI)
- 로드맵: 2~5교시 예고 (이론은 PPT, 실습은 셀프스터디)

## 신규 투입되는 '이론' 정리 (이번에 두꺼워지는 알맹이)
- 변수의 종류(범주형/수치형) — 분석·차트 선택의 기초
- 기본 통계: 중심값(평균/중앙값/최빈값), 산포(범위/분산/표준편차), 분포·치우침
- 표본과 편향 / 결측 / **상관 ≠ 인과**
- 데이터 보는 2단계: 전체 보기(요약통계) → 나눠 보기(세분화)
- 시각화 차트 4종 '언제 쓰나' (개념)
- 기업 데이터 파이프라인: 수집 → 처리 → 분석 → 활용

## 도식·차트 목록 (네이티브 도형 / 이미지)
- (유지) 평균함정·전투기·버클리·비트코인 차트(이미지)
- (신규) 변수 종류 분류 도식 / 중심값·산포 개념 도식 / 분포(치우침) 히스토그램 이미지
- (신규) 상관≠인과 도식(아이스크림-익사-여름) / 데이터 파이프라인 4단계 도식
- (유지·압축) 패러다임 4단계 / 에이전트 파이프라인 / 역할 이동 FROM→TO
