## 1. 왜 정제(cleaning)가 필요할까?

교과서의 데이터는 깔끔합니다. 하지만 **현실의 데이터는 거의 항상 지저분**합니다.

- 빈 칸이 있고 (나이를 안 적은 승객),
- 숫자여야 할 게 글자로 들어있고 (`"19,500"` 처럼 쉼표 포함),
- 같은 뜻인데 다르게 적혀 있습니다 (`서울` / `서울특별시`).

> **"Garbage In, Garbage Out (쓰레기가 들어가면 쓰레기가 나온다)."**
> 아무리 좋은 분석·AI도, 입력 데이터가 엉망이면 결과도 엉망입니다. 그래서 분석가는 **시간의 절반 이상을 데이터 정제**에 씁니다. 화려하진 않지만 가장 중요한 기본기예요.

이 장에서는 가장 자주 만나는 세 가지를 다룹니다: **① 결측치 ② 타입 ③ 그룹 요약.**

## 2. 결측치(missing value) — 비어있는 값 다루기

**결측치 = 값이 비어 있는 칸.** 판다스에서는 보통 `NaN`(Not a Number)으로 표시됩니다.

### 2-1. 찾기
```python
df.isnull().sum()      # 열마다 빈 칸이 몇 개인지 세기
```
2장에서 본 `info()`로도 보이지만, `isnull().sum()`이 더 또렷하게 "어느 열에 몇 개"를 알려줍니다.

### 2-2. 어떻게 할까 — 두 가지 길
빈 칸을 만나면 선택지는 크게 둘입니다.

| 방법 | 명령 | 언제 쓰나 | 주의 |
|---|---|---|---|
| **버리기** | `df.dropna()` | 빈 칸이 적거나, 그 행을 못 믿을 때 | 데이터가 줄어듦 (너무 많이 버리면 위험) |
| **채우기** | `df.fillna(값)` | 빈 칸을 합리적인 값으로 메울 때 | 무슨 값으로 채울지 신중히 |

```python
df.dropna()                       # 빈 칸이 하나라도 있는 행을 통째로 삭제
df["age"].fillna(df["age"].mean())  # 나이의 빈 칸을 '평균 나이'로 채우기
```

> 💡 **정답은 없습니다.** "나이를 평균으로 채울까, 그 행을 버릴까?"는 상황 판단입니다. 바로 이 판단이 — 1장에서 말한 — *사람의 몫*이에요.

## 3. 타입 변환 — 글자를 숫자로

컴퓨터는 **글자(object)** 와 **숫자(int, float)** 를 다르게 봅니다. 숫자처럼 *보여도* 글자로 저장돼 있으면 평균조차 못 구합니다.

대표 사례가 바로 **아파트 실거래가**입니다. 거래금액이 `"19,500"`처럼 **쉼표가 든 글자**로 들어있죠.

```python
# (1) 쉼표 제거 → (2) 숫자로 변환
df["거래금액(만원)"] = df["거래금액(만원)"].str.replace(",", "").astype(int)
```
- `.str.replace(",", "")` : 글자에서 쉼표를 지웁니다. `"19,500"` → `"19500"`
- `.astype(int)` : 글자를 *정수*로 바꿉니다. `"19500"` → `19500`

이제야 평균·최대 같은 계산이 됩니다.
```python
df["거래금액(만원)"].mean()    # 평균 거래금액
```

| 타입 이름 | 뜻 | 예 |
|---|---|---|
| `object` | 글자(문자열) | `"서울"`, `"19,500"` |
| `int64` | 정수 | `19500`, `3` |
| `float64` | 소수 | `39.08`, `7.5` |

> 📌 **순서가 중요**: 글자 상태에선 못 더합니다. *먼저 정제(쉼표 제거) → 타입 변환 → 그다음 계산.*

## 4. 그룹별 요약 — `groupby`

분석에서 가장 강력한 무기 중 하나입니다. **"그룹으로 나눠서 비교하기."**

"전체 평균"보다 **"그룹별 평균"** 이 훨씬 많은 걸 말해줍니다.
- 전체 생존율보다 → *객실 등급별* 생존율
- 전체 매출보다 → *요일별* 매출

```python
df.groupby("pclass")["survived"].mean()
#   ↑ 객실등급으로 묶어서   ↑ 생존여부의   ↑ 평균
```
이 한 줄이 "1등석/2등석/3등석 각각의 생존율"을 한 번에 내줍니다. (결과: 1등석 63%, 3등석 24% — 등급이 생존을 갈랐네요.)

**`groupby`의 3단 구조**를 기억하세요:
```
df.groupby("기준 열")["볼 열"].요약함수()
        ↓             ↓          ↓
     무엇으로 묶나   무엇을 보나   어떻게 요약하나(mean/sum/count…)
```

### 단짝 친구 `value_counts()`
그룹이 아니라 **"각 값이 몇 번 나오나"** 가 궁금하면:
```python
df["embarked"].value_counts()   # 탑승 항구별 인원 수
```

## 5. 정렬과 필터 — 보고 싶은 것만, 순서대로

```python
df.sort_values("fare", ascending=False)   # 요금 높은 순으로 정렬
df[df["age"] < 18]                          # 18세 미만 행만 골라보기
```
- `sort_values` : 특정 열 기준 정렬 (`ascending=False`면 큰 값부터).
- `df[조건]` : 조건을 만족하는 행만 *걸러내기*(필터링). "미성년자만 보자."

> 💡 `df["age"] < 18`은 각 행에 대해 참/거짓을 만들고, `df[...]`가 *참인 행만* 남깁니다. 자주 쓰니 눈에 익혀두세요.

## 📎 3장 요약 한 장

- 현실 데이터는 지저분하다 → **정제는 분석의 절반.** "쓰레기가 들어가면 쓰레기가 나온다."
- **결측치**: `isnull().sum()`(찾기) → `dropna()`(버리기) / `fillna()`(채우기). 무엇을 택할지는 *판단*.
- **타입 변환**: 글자(`object`)는 계산 불가. `.str.replace(",", "")` → `.astype(int)`. *정제 먼저, 계산은 나중.*
- **그룹 요약**: `df.groupby("기준")["볼 열"].mean()` — 전체보다 그룹별 비교가 강하다. `value_counts()`는 값별 개수.
- **정렬·필터**: `sort_values()`, `df[조건]`.
