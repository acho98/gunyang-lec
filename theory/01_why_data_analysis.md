# 1장. 데이터 분석, 왜 그리고 어디로

> **이 장의 목표**
> - 우리가 *왜* 데이터를 분석하는지 실생활 사례로 체감한다.
> - 분석의 도구가 **엑셀 → 통계 → 머신러닝 → LLM·에이전트**로 어떻게 변해왔는지 큰 그림을 잡는다.
> - 그리고 가장 중요한 한 문장을 가슴에 새긴다:
>   **"도구는 변해도, 좋은 질문을 던지고 결과를 해석하는 힘은 사람의 몫이다."**
>
> 이 장은 **이론 위주**입니다. 코드는 거의 없고, 강사의 짧은 시연을 *구경*하며 "분석이란 이런 거구나"를 느끼는 시간입니다.

---

## 1. 왜 데이터를 분석할까?

여러분은 오늘 하루에도 수십 번 **데이터 분석의 결과물**을 사용했습니다. 스스로 의식하지 못했을 뿐입니다.

- **유튜브·넷플릭스**: "다음에 볼 영상"을 추천해 줍니다. → 수많은 사람의 시청 기록을 *분석*해서, 여러분이 좋아할 확률이 높은 걸 고른 것.
- **배달앱**: "이 가게 어때요?", "보통 30분 걸려요" → 과거 주문·배달 데이터를 *분석*한 결과.
- **지도앱**: "지금 출발하면 7시 12분 도착" → 도로의 실시간·과거 교통 데이터를 *분석*한 예측.
- **쇼핑몰**: "이 상품을 산 사람들이 함께 본 상품" → 구매 기록을 *분석*해 묶어준 것.

이 모든 것의 공통점은 하나입니다.

> **데이터 분석 = 쌓인 데이터에서 "쓸모 있는 답"을 끄집어내는 일.**

조금 더 풀면, 데이터 분석은 보통 이런 질문에 답하기 위해 합니다.

| 질문의 종류 | 예시 | 한 줄 설명 |
|---|---|---|
| **무슨 일이 있었나?** | 지난달 매출이 왜 줄었지? | 과거를 *설명* |
| **왜 그럴까?** | 어떤 고객이 떠났을까? | 원인을 *탐색* |
| **앞으로 어떻게 될까?** | 다음 달 주문량은? | 미래를 *예측* |
| **그래서 뭘 해야 하나?** | 누구에게 쿠폰을 줄까? | 행동을 *결정* |

### 비즈니스는 왜 분석에 진심일까

기업 입장에서 데이터 분석은 "있으면 좋은 것"이 아니라 **생존**입니다. 예를 들어,

- **고객을 나눠서 본다(고객 세분화).** 모든 손님을 똑같이 대하지 않고, "자주 오는 단골 / 한 번 오고 안 오는 손님 / 떠나려는 손님"으로 나눠 각각 다르게 대응합니다.
- **떠날 손님을 미리 안다(이탈 예측).** 통신사가 "이 고객은 곧 해지할 것 같다"를 미리 알아채면, 떠나기 전에 혜택을 줘서 붙잡을 수 있습니다.

> 💡 이런 게 바로 데이터 분석이 **돈과 직결**되는 지점입니다. 좋은 질문 하나가 회사의 매출을 바꾸기도 합니다.

### 강사의 현장 이야기 — 금융·부동산 데이터

저는 **핀테크(금융 기술)** 분야에서 데이터를 다뤄 왔습니다. 거창해 보이지만, 출발은 여러분이 지금 배우는 것과 똑같습니다. 표(데이터)를 불러와서, 들여다보고, 그림으로 그려서 패턴을 찾는 일이죠.

- **비트코인 시세**: 매일의 가격을 그래프로 그려 보면, 급등·급락의 흐름이 눈에 보입니다.
- **금(Gold) 시세**: 경제가 불안할 때 사람들이 금을 찾는지, 가격 흐름으로 살펴봅니다.
- **아파트 실거래가**: 국토교통부가 공개하는 *실제 거래 기록*을 분석하면 "어느 동네 가격이 오르나"가 보입니다.

이 데이터들은 **이번 과정에서 실제로 만져볼 자료**입니다(1장 시연, 그리고 3장 실습). 중요한 건 — 이 모든 분석이 **여러분이 앞으로 4시간 동안 배울 기본기**(불러오기 → 정제 → 시각화)에서 시작한다는 점입니다.

## 2. 분석의 도구는 어떻게 변해왔나 — 4단계 큰 그림

데이터 분석의 *본질*("데이터에서 답을 찾는다")은 100년 전이나 지금이나 똑같습니다. 하지만 **도구**는 계속 변해왔습니다. 큰 흐름을 4단계로 봅시다.

```
 1단계            2단계             3단계               4단계
 손·엑셀    →    통계·BI     →    빅데이터·ML    →    LLM·에이전트
 (직접 계산)     (요약·차트)      (예측·추천)        (대화로 분석)
```

### 1단계 — 손과 엑셀: "직접 세고 더한다"
표에 숫자를 적고, 계산기나 엑셀로 합계·평균을 냅니다. 작은 데이터엔 지금도 훌륭한 도구입니다. 하지만 데이터가 수십만 줄이 되면 엑셀은 버거워집니다.

### 2단계 — 통계와 BI: "요약하고 그림으로 본다"
평균·비율 같은 **통계**로 데이터를 요약하고, 대시보드(BI 도구)로 한눈에 봅니다. "지난 분기보다 매출이 늘었나?" 같은 질문에 답하죠. *여러분이 2~4장에서 배울 게 바로 이 단계의 핵심 기본기입니다.*

### 3단계 — 빅데이터와 머신러닝: "기계가 패턴을 학습한다"
데이터가 너무 많고 복잡해지자, 사람이 규칙을 일일이 정하는 대신 **기계가 데이터에서 규칙을 스스로 학습**하게 만들었습니다(머신러닝). 넷플릭스 추천, 스팸 메일 자동 분류, 얼굴 인식이 모두 이 단계의 산물입니다.

### 4단계 — LLM과 에이전트: "말로 분석을 시킨다" (지금 여기)
2023년 이후, **ChatGPT 같은 LLM(거대 언어 모델)** 이 등장하면서 또 한 번 바뀌고 있습니다. 이제는 **사람 말로** "이 데이터에서 뭐가 보여?"라고 물으면, AI가 코드를 대신 써서 분석해 주기도 합니다.

> 한 줄 정리: **도구는 점점 "더 쉽고, 더 똑똑하게" 진화**해 왔습니다. 그런데 — 뒤에서 보겠지만 — 도구가 똑똑해질수록 오히려 **사람의 역할이 더 중요해집니다.**

## 3. LLM 시대의 데이터 분석 — ChatGPT에게 물어보는 분석

요즘 가장 큰 변화는 **ChatGPT** 같은 AI에게 *말로* 분석을 부탁할 수 있게 된 것입니다.

예전에는 데이터를 분석하려면 코드를 직접 한 줄 한 줄 써야 했습니다. 지금은 이렇게 할 수 있습니다.

> **여러분**: (엑셀 표를 올리며) "이 판매 데이터에서 어떤 제품이 가장 잘 팔렸고, 요일별로 차이가 있는지 알려줘. 그래프도 그려줘."
>
> **ChatGPT**: (스스로 코드를 작성·실행해서) "주말에 A제품 판매가 평일보다 40% 높습니다. 아래 그래프를 보세요…"

실제로 ChatGPT에는 데이터를 올리면 분석·그래프를 만들어 주는 기능이 있고(흔히 '데이터 분석' 기능), 전 세계에서 매주 수억 명이 사용합니다. **이 수업의 1장 시연에서 직접 보여드릴 겁니다.**

이게 의미하는 바는 큽니다.

- **장벽이 낮아졌다**: 코드를 몰라도 데이터에 질문을 던질 수 있게 됨.
- **속도가 빨라졌다**: 몇 시간 걸리던 집계·그래프를 몇 분 만에.

> ⚠️ **하지만 만능은 아닙니다.** AI는 여러분이 *잘못된 질문*을 하면 *그럴듯하지만 틀린 답*을 자신 있게 내놓기도 합니다. 또 회사의 정형 데이터(매출 예측 같은)는 여전히 전용 분석 방법이 더 정확한 경우가 많습니다. **AI의 답을 검증하고 해석하는 건 결국 사람의 몫**입니다 — 이 이야기는 곧 다시 합니다.

## 4. 'AI 에이전트'란? — 인식 수준으로만

요즘 뉴스에 **'AI 에이전트(agent)'** 라는 말이 자주 나옵니다. 초급 단계에서는 딱 이 정도만 알면 충분합니다.

> **에이전트 = "시키면 여러 단계를 알아서 척척 해내는 AI 비서".**

지금까지의 AI가 *질문에 답하는* 수준이었다면, 에이전트는 **목표를 주면 여러 단계를 스스로 밟아 일을 끝까지** 처리합니다.

- 비유하자면, 예전 AI가 "레시피를 알려주는 요리책"이라면, 에이전트는 **"재료를 사다가 직접 요리까지 해주는 보조 요리사"** 에 가깝습니다.
- 데이터 분석에서는: "이번 달 판매 데이터를 정리해서, 분석하고, 리포트까지 만들어 줘"라고 하면 — 데이터를 찾고 → 분석하고 → 그래프를 그려 → 보고서를 만드는 여러 단계를 **한 번에** 처리하려 시도합니다.

*(곁가지: ChatGPT 외에도 여러 사이트를 넘나들며 작업하는 브라우저형 'Atlas', 작업을 통째로 대신 굴리는 'Manus' 같은 도구들이 2025년에 등장했습니다 — 이런 게 있다는 정도만 알아두세요.)*

> 📌 초급 단계의 결론: **에이전트의 기술적 원리(토큰, 내부 구조 등)는 지금 몰라도 됩니다.** "AI가 점점 더 많은 단계를 알아서 해주는 쪽으로 간다" — 이 *방향성*만 기억하세요.

## 5. 그래서, 핵심 메시지 — 도구는 변해도 사람이 할 일은 남는다

도구는 엑셀에서 ChatGPT, 에이전트까지 점점 똑똑해졌습니다. 그러면 "이제 분석은 AI가 다 하니까, 사람은 배울 필요 없는 것 아닌가?" — 정반대입니다.

자동화될수록, **사람만 할 수 있는 두 가지**가 더 빛납니다.

1. **좋은 질문을 던지는 힘.**
   AI는 "무엇을 물을지"는 정해주지 않습니다. "우리 손님이 왜 떠날까?", "이 데이터로 뭘 알아낼 수 있을까?" — *질문을 설계하는 건 사람*입니다. 질문이 엉성하면 AI의 답도 엉성합니다.

2. **결과를 해석하고 판단하는 힘.**
   AI가 그래프와 숫자를 내놓아도, "이게 말이 되는가? 진짜 의미가 뭔가? 이걸로 무슨 결정을 내릴까?"를 판단하는 건 사람입니다. AI는 가끔 자신 있게 틀립니다. 그걸 **알아채려면 기본기가 있어야** 합니다.

> **이 과정의 한 문장**
> ## "도구는 변해도, 좋은 질문을 던지고 결과를 해석하는 힘은 사람의 몫이다."

바로 이래서, 우리는 앞으로 4시간 동안 **분석의 기본기**를 배웁니다.

- **2장**: 데이터를 불러와 들여다보기
- **3장**: 지저분한 데이터를 깨끗이 정제하기
- **4장**: 그림으로 인사이트 뽑기
- **5장**: 배운 걸 종합하고, 다음 공부 길을 잡기

이 기본기가 있어야, 똑똑한 AI 도구를 **부려 쓰는 사람**이 됩니다. 기본기가 없으면, AI가 주는 답이 맞는지 틀리는지조차 모른 채 끌려다니게 됩니다.

## 📎 1장 요약 한 장

- 데이터 분석은 **쌓인 데이터에서 쓸모 있는 답을 찾는 일** — 우리는 매일 그 결과를 쓰고 있다(추천·예측·지도).
- 도구는 **엑셀 → 통계·BI → 머신러닝 → LLM·에이전트**로 진화해 왔다. 점점 쉽고 똑똑해진다.
- **ChatGPT**로 말로 분석을 시키는 시대지만, AI는 만능이 아니다(자신 있게 틀리기도 한다).
- **에이전트**는 "여러 단계를 알아서 해주는 AI 비서" — 방향성만 알면 충분(원리는 나중에).
- 그래서 더 중요해진 건 **사람의 두 가지 힘**: ① 좋은 질문 ② 결과 해석.
- **"도구는 변해도, 질문과 해석은 사람의 몫"** — 이게 우리가 기본기를 배우는 이유다.
