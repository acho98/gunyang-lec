# 초급 데이터 분석 과정 — 제작 계획서 (PLAN)

> 대상: 대학생 (프로그래밍·통계 사전지식 거의 없음) · 분량: 5시간(5세션 × 1h)
> 성격: **이론 위주 + 짧은 핵심 실습 + 정답 제공 + 셀프스터디 발판**
> 본 문서는 **1단계 산출물**입니다. 확인·수정 후 2단계(본 제작)로 진행합니다.

## 0. 한눈에 보기 (TL;DR)

- 5시간은 빠듯하다. **개념·큰 그림·사고방식**을 심는 데 집중하고, 실습은 짧게, 정답을 함께 줘서 "집에서 혼자 더 파는 발판"을 쥐여주는 설계.
- 원본 4개 폴더는 대부분 **중급 이상**(ML·비즈니스 분석)이라 그대로는 못 쓴다. **판다스/EDA/시각화 기초 부분만 발췌**해 초급 눈높이로 재가공한다.
- 실습 데이터는 **고전(titanic·tips·penguins) + 최신(World Happiness 최신판·한국어 게임리뷰)** 을 섞는다. 안정성을 위해 **seaborn 내장 데이터(다운로드 불필요)** 를 백본으로 삼아 "모든 코드가 항상 실행됨"을 보장한다.
- **5시간 안에 실습을 하나하나 다 돌리는 건 불가능**하다는 걸 전제로 한다. 수업 중엔 강사가 **핵심 셀 몇 개만 시연·같이 실행**하고, 노트북 자체는 **셀프스터디용 완전체**로 만든다. 셀마다 🔵수업 / 🟡셀프로 마킹하고 **정답을 전부 제공**해 집에서 대조하게 한다.
- **fintech 폴더(강사 본인 제작)** 는 중급~고급 위주라 본류 실습엔 안 넣고, **1장 동기부여(현장감·신뢰도 ↑)** 와 **5장 심화 로드맵**에 쓴다. 아파트 실거래가만 3장 "한국 공공데이터 정제" 선택 데모로 발췌.
- **1장은 트렌드가 생명** → 웹 검색으로 LLM·AI 에이전트 시대의 분석 변화를 최신화하되, **인식·맥락 수준**으로만 다룬다(토큰·RAG·아키텍처 등 디테일 금지).

## 1.5 실습 운영 방식 & 수업 설계 철학 (중요)

**전제: 5시간 안에 모든 실습을 수업 중 하나하나 다 돌리는 것은 불가능하다.** 그래서 실습을 "수업 중 완성"의 대상으로 삼지 않는다. 대신 다음 방식으로 운영한다.

- **수업 중**: 강사가 **핵심 셀 몇 개만 시연하거나 학생과 같이 실행**한다. 흐름과 개념을 잡는 게 목적이지, 모든 줄을 타이핑시키지 않는다.
- **실습 노트북 = 셀프스터디용 완전체**: 노트북 하나만 있으면 집에서 혼자 끝까지 갈 수 있게 설명·코드·정답을 빠짐없이 담는다. 수업에서 못 다룬 셀도 노트북 안에서 자기완결적으로 이해되도록 작성.
- **셀 단위 마킹**: 각 실습 노트북 안에서 셀을 시각적으로 구분한다.
  - 🔵 **수업** — 강사가 수업 중 시연·같이 실행할 핵심 셀(장당 3~5개 내외).
  - 🟡 **셀프** — 집에서 직접 해볼 셀(빈칸/TODO 포함).
  - 마킹은 각 셀 상단 마크다운에 `> 🔵 수업` / `> 🟡 셀프` 배지로 표기.

## 2. 목차 (커리큘럼) — 5세션 상세

> 각 세션의 실습 설명에 **🔵 수업 셀 / 🟡 셀프 셀** 비중을 함께 표기한다(위 1.5 운영 방식 참조).

> 비중 표기: 이론:실습. 초급·이론위주 원칙상 전체적으로 이론에 무게.

### 1장 (60분) — 데이터 분석, 왜 그리고 어디로 〔트렌드·웹서치 보강〕
- **학습목표**: 데이터 분석을 *왜* 하는지 체감하고, 도구가 LLM·에이전트로 어떻게 변하는지 *맥락*을 잡는다. "도구는 변해도 **좋은 질문과 해석은 사람 몫**"을 가슴에 새긴다.
- **이론:실습 = 90:10** (실습은 손이 아니라 눈/시연 위주)
- 내용: ① 왜 분석하나(넷플릭스 추천·유튜브 알고리즘·기업의 고객 이해=RFM/AARRR 쉬운 버전) ② **금융·핀테크 현장 사례**(강사 본인 fintech 자료 — 비트코인·금값·부동산 실거래가를 데이터로 들여다본 실제 경험 → 현장감·신뢰도↑) ③ 패러다임 변화 4단계 ④ LLM 시대의 분석(ChatGPT로 데이터에 질문) ⑤ 에이전트 개념·친숙 사례(인식 수준) ⑥ 핵심 메시지 → 4시간 기초의 동기.
- **실습 운영 🔵수업 위주(거의 시연)**: 🔵 강사가 fintech `collect_data_01`로 "비트코인·금값을 단 몇 줄로 가져와 그래프로 본다"를 시연 + ChatGPT/노트북 AI에 작은 표를 주고 "뭐가 보여?"를 *함께 구경*. 🟡 셀프 = 생각해볼 질문 3개(코딩 없음). 손코딩 거의 없음.
- **데이터셋**: 시연용 미니 표(직접 제작) + fintech 비트코인/금값 시세(가격 추이 차트) + 실제 데이터 생김새 예시(crm Online Retail·국토부 아파트 실거래가 캡처).

### 2장 (60분) — 분석 환경 + 데이터 들여다보기 (주피터·판다스 기초, EDA 시작)
- **학습목표**: 주피터 노트북·판다스가 뭔지 이해하고, 데이터를 **불러와 첫인상을 읽는다**(모양·요약·미리보기).
- **이론:실습 = 50:50**
- 내용: ① 주피터/파이썬/판다스가 뭔가(엑셀 ↔ DataFrame 비유) ② 표 데이터 구조(행=관측치, 열=변수) ③ 불러오기·살펴보기 5종 세트(`head`,`shape`,`info`,`describe`,`columns`).
- **실습(짧게)**: titanic 불러와 5종 세트로 "이 배에 누가 탔나" 첫인상 읽기. → 🔵 수업 셀: 불러오기·`head`·`info`(3셀 시연) / 🟡 셀프 셀: `describe`·`columns`로 직접 읽기(빈칸+정답).
- **데이터셋**: **titanic**(seaborn) — 고전·친숙(영화). 보조: prd `mlb.csv`(개념 예시).

### 3장 (60분) — 데이터 정제·가공 (결측치·타입·그룹핑)
- **학습목표**: 현실 데이터는 더럽다는 걸 알고, **결측치·타입·그룹 집계**의 기본기를 익힌다.
- **이론:실습 = 50:50**
- 내용: ① 왜 정제하나(쓰레기 들어가면 쓰레기 나온다) ② 결측치 다루기(`isnull`/`dropna`/`fillna`) ③ 타입 변환(`astype`) ④ 그룹별 요약(`groupby`/`value_counts`) ⑤ 정렬·필터.
- **실습(짧게)**: titanic 결측 age 채우기 + 객실등급별 생존율 `groupby`. → 🔵 수업 셀: `isnull`·`fillna`·`groupby` 핵심 3~4셀 시연 / 🟡 셀프 셀: `astype`·`value_counts`·정렬 직접(빈칸+정답). (🟡 선택 심화 데모) 한국어 게임리뷰 또는 **fintech 아파트 실거래가**로 "지저분한 한국 데이터 정제"(거래금액 쉼표 제거→숫자 변환) 맛보기.
- **데이터셋**: **titanic**(결측 age/deck) + **tips**(groupby). 모던 보강(선택): 게임리뷰 CSV 또는 fintech `apt_2021xx_01.csv`(국토부 실거래가).

### 4장 (60분) — 시각화로 인사이트 뽑기
- **학습목표**: 숫자를 그림으로 바꿔 **패턴을 한눈에** 읽는다. 어떤 차트를 *언제* 쓰는지 감을 잡는다.
- **이론:실습 = 40:60** (시각화는 손으로 봐야 와닿음)
- 내용: ① 왜 시각화하나(같은 데이터도 표보다 그림) ② 차트 4종 언제 쓰나(막대=비교, 히스토그램=분포, 산점도=관계, 박스=그룹비교) ③ 한글 폰트 깨짐 처리.
- **실습(짧게)**: penguins 산점도(부리 길이 vs 깊이, 종별 색), happiness 막대·상관. seaborn 한 줄 차트 위주. → 🔵 수업 셀: 한글폰트 설정 + 산점도·막대 2~3셀 시연 / 🟡 셀프 셀: 히스토그램·박스플롯·상관히트맵 직접(빈칸+정답).
- **데이터셋**: **penguins**(seaborn, 모던·예쁨) + **World Happiness 최신**(동봉 CSV) + tips.

### 5장 (60분) — 미니 프로젝트 + 셀프스터디 로드맵
- **학습목표**: 배운 흐름(불러오기→정제→그룹핑→시각화→해석)을 **한 데이터로 끝까지** 꿰고, **앞으로 뭘 더 공부할지** 지도를 손에 쥔다.
- **이론:실습 = 40:60**(앞 절 프로젝트) + 마무리 로드맵.
- **실습 운영**: 미니 프로젝트는 🔵 수업에서 강사가 "질문 정하기→데이터 고르기→첫 셀"까지 흐름만 같이 잡고(2~3셀), 🟡 나머지 전 과정은 셀프스터디로(정답 노트북 전체 제공). 집에서 한 번 더 완주하는 게 핵심.

- **데이터셋**: **World Happiness 최신** 또는 **penguins**(종합). 심화 예시 참고: fintech 비트코인·금값·AirPassengers(시계열).

## 3. 작성 관점 설명 (왜 이렇게 구성했나)

**(1) "초급·이론위주" 원칙**
원본 4폴더는 대부분 중급 이상이라 그대로 못 쓴다. **판다스/EDA/시각화 기초만 외과적으로 발췌**하고, ML·연관규칙·이탈예측·코호트·A/B는 **과감히 제외**했다. 5시간 안에 "다 가르치려는 욕심"을 버리고, **개념·큰 그림·사고방식**에 무게를 실었다. 각 장 이론:실습 비중도 손코딩보다 이해를 앞세웠다.

**(2) "짧은 실습 + 정답 제공 + 셀프스터디"**
실습은 장당 1~2개, **빈칸/TODO 배포본 + 정답본**을 분리한다. 목적은 "수업 중 완성"이 아니라 "수업에서 흐름을 잡고 집에서 정답 보며 스스로 파기". 5장 로드맵이 그 발판을 명시한다.

**(3) "순서"의 논리 — 분석의 실제 흐름 그대로**
1장(왜/어디로, 동기) → 2장(불러와 보기) → 3장(정제) → 4장(시각화) → 5장(종합+다음 단계). 이는 **실제 분석가가 일하는 순서**(질문→수집→정제→탐색·시각화→해석)와 같아, 학생이 "도구 나열"이 아니라 **하나의 이야기**로 기억하게 한다.

**(4) "1장은 트렌드가 생명"**
원본에 없던 LLM·에이전트 흐름을 웹 검색으로 최신화하되, **초급 눈높이로 소화**해 인식·맥락 수준만 다룬다(토큰·RAG·아키텍처 금지). 친숙 사례(ChatGPT 에이전트 모드·추천 알고리즘) 위주로, **"도구는 변해도 좋은 질문·해석은 사람 몫"** 이라는 메시지를 깔아 나머지 4시간 학습의 동기로 연결한다. 5장에서 이 메시지를 다시 닫아 **수미상관** 구조를 만든다.

**(5) "데이터셋 — 고전과 최신의 균형 + 무조건 실행되게"**
개념 설명엔 **검증된 고전**(titanic·tips·penguins)을, 흥미 유발엔 **최신·친숙**(World Happiness 최신·한국어 게임리뷰)을 섞는다. 안정성을 위해 **seaborn 내장 로드를 기본**으로 삼아 다운로드·인증·경로 문제를 없애고, 동봉 CSV는 상대경로로 고정해 **"모든 코드가 실행된다"** 는 약속을 지킨다.

**(6) "실습은 다 못 돌린다는 현실을 설계에 박았다"**
5시간 안에 모든 실습을 수업 중 완료하는 건 불가능하다. 그래서 실습을 **셀프스터디용 완전체**로 만들고, 수업 중엔 🔵 핵심 셀만 시연한다. 🔵수업/🟡셀프를 셀 단위로 마킹하고 정답을 전부 제공해, "수업에서 흐름을 잡고 집에서 정답 보며 완주"하는 구조를 노트북 자체에 내장했다(§1.5). 이는 욕심내지 않고 개념·발판에 집중하라는 설계 철학의 실행 장치다.

**(7) "fintech는 강사의 무기 — 본류 대신 양 끝에 배치"**
강사 본인이 만든 fintech 자료는 **현장감·신뢰도**가 가장 큰 자산이다. 다만 수집·시계열·이상탐지는 중급~고급이라 초급 본류 실습에 넣으면 무너진다. 그래서 **1장 동기부여(이 강의의 첫 인상)** 와 **5장 심화 로드맵(이 강의의 출구)** 이라는 양 끝에 배치해, 초급 난이도를 지키면서도 "실무하는 사람이 가르친다"는 신뢰와 "더 가면 이런 게 있다"는 동경을 동시에 준다. 초급 발췌는 아파트 실거래가 정제 데모(3장) 하나로 절제했다.

**(8) "원본은 읽기전용 참조"**
analytic/crm/prd/python_ml/fintech는 **수정하지 않고** 소스로만 참조한다. 산출물은 전부 `intro_data_analysis/` 안에 새로 만든다.
